# Project Chimera — Glossary

This glossary defines canonical terms used across all specifications.
Terms are case-sensitive and must not be redefined elsewhere.

---

**Agent**  
A persistent autonomous AI entity operating within the Chimera system.

**Autonomous Influencer**  
An Agent designed to operate a public-facing digital persona.

**Orchestrator**  
The central control plane responsible for agent lifecycle, coordination,
and governance.

**Planner**  
An agent role responsible for decomposing goals into executable tasks.

**Worker**  
An agent role responsible for executing a single atomic task.

**Judge**  
An agent role responsible for validating outputs, enforcing policy,
and escalating to humans when required.

**Skill**  
A bounded, reusable capability with a strict input/output contract.
Skills are owned by Chimera and invoked by agents.

**Tool**  
An external capability exposed to agents via MCP.
Tools are not owned or implemented by Chimera agents.

**Model Context Protocol (MCP)**  
A standardized protocol that governs how agents access external
resources, tools, and prompts.

**Human-in-the-Loop (HITL)**  
A governance mechanism requiring human review or approval
before certain agent actions may proceed.

**Specification**  
A formal, version-controlled description of intent.
Specifications are the system’s source of truth.

**Task**  
A discrete unit of work generated by a Planner and executed by a Worker.

**Audit Log**  
An immutable record of agent decisions, actions, and outcomes.

---

Any term not defined here must be added before use.
